### 날짜별 TIL 구성

각 수업일에는 아래 두 파일이 미리 준비되어 있습니다.

```text
YYYY-MM-DD_daily_til.ipynb  # 작성하고 실행하는 원본
YYYY-MM-DD_daily_til.html   # 게시·공유용 최종본
```

노트북은 다음 흐름으로 작성합니다.

```text
오늘의 핵심
→ 주제별 학습 기록
→ 실습 코드와 결과
→ 배운 점과 다음 액션
```

- 하루에 노트북 1개, HTML 1개만 사용합니다.
- 오전·오후 주제가 같으면 하나의 학습 기록으로 합칩니다.
- 같은 날 주제가 다르면 노트북 안에서 주제별 소제목으로 나눕니다.
- 코드가 없는 날에는 글 중심으로 작성하고 `실습 코드` 영역을 생략할 수 있습니다.
- 작성이 끝난 노트북은 같은 폴더의 HTML로 정리합니다.

### 기록 공간의 역할

| 공간 | 역할 |
|---|---|
| Notion | 수업 중 배운 내용·생각·질문·다음 액션을 빠르게 기록 |
| VS Code / Jupyter | 공개 가능한 설명과 실습 코드·결과를 하나의 TIL로 정리 |
| HTML | 완성된 노트북을 매일 게시할 수 있는 형태로 저장 |
| 주차 README | 해당 주의 날짜별 TIL과 작성 상태를 확인 |

같은 글을 여러 곳에 반복해서 작성하지 않습니다. Notion의 메모와 실습 코드를 하루의 노트북으로 합치고, 그 노트북에서 HTML을 만듭니다.

### 공개 원칙

이 저장소에는 제가 이해하고 재구성한 학습 내용과 직접 작성한 코드만 남깁니다.

- 강의 교안·유료 자료·과제 정답을 그대로 옮기지 않습니다.
- 강사와 수강생의 이름·연락처·발언 등 개인정보를 기록하지 않습니다.
- 수업 시간, 내부 공지, 협업 링크와 접속 정보를 공개하지 않습니다.
- 인증키와 계정 정보, 공개할 수 없는 원본 데이터는 커밋하지 않습니다.
- 기업명·매출·광고비 등 실습 자료의 민감한 정보는 제거하거나 일반화합니다.
- HTML은 게시용 로컬 파일로 관리하며 GitHub에는 올리지 않습니다.
